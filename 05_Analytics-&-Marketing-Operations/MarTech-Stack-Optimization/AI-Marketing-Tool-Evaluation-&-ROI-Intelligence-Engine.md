## Integration Tips

**HubSpot:** Export your current tool stack from HubSpot's Marketing Hub settings → Connected Apps. Paste the list directly into the "CURRENT TECH STACK" field to auto-populate integration compatibility analysis.

**Salesforce:** Pull your "Opportunity Source" report for the last 12 months to populate the revenue-per-conversion figures in the Value Quantification Model. This makes ROI projections defensible to the CFO.

**Notion / Confluence:** Create a standing "AI Tool Evaluation" database. Run this prompt for every tool, then paste outputs as structured pages. Tag by verdict (Buy / Trial / Pass) and review quarterly to catch tools whose roadmap has matured to reconsider.

**Google Sheets:** Use the TCO and value outputs to build a comparison matrix. Paste the scorecard numbers for 3-4 competing tools into a spreadsheet for side-by-side stack ranking — ideal for presenting 2-3 finalists to leadership.

**Zapier / Make:** When a vendor sends a demo follow-up email, trigger a Zap that extracts tool name and pricing from the email and pre-populates the Quick Copy-Paste prompt. Reduces evaluation setup time to under 5 minutes.

**Procurement / Legal:** Feed the Procurement Intelligence section output directly into your vendor negotiation doc. Use the specific discount ranges and contract term pushbacks as your opening negotiation script.

## Troubleshooting

**Issue: ROI estimates feel too optimistic or too low.**
Solution: The model defaults to vendor case study benchmarks with a 30% haircut. If you have internal data (e.g., your current content cost per piece, your actual conversion rates), substitute real numbers into the Advanced Version's value driver fields. Real inputs always outperform generic benchmarks.

**Issue: The AI returns a generic evaluation that doesn't address our specific stack.**
Solution: The most common cause is an underspecified stack description. Don't just list tool names — include the tier (e.g., "HubSpot Marketing Hub Pro, not Enterprise") and how each is used (e.g., "Salesforce for CRM but we only use Accounts and Opportunities, not Leads"). Integration analysis improves dramatically with specificity.

**Issue: Recommendation keeps coming back as "Trial First" even when we need a buy/pass decision.**
Solution: Add this line to the end of either prompt version: "We cannot run a trial due to [reason: data security policy / no eng bandwidth / Q4 freeze]. Force a BUY or PASS recommendation based on existing evidence only, and increase confidence interval accordingly."
