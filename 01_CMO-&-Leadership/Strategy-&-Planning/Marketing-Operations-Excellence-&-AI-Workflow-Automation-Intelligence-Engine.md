## Integration Tips

**Salesforce:**
Use Salesforce Flow (formerly Process Builder) to build AI-augmented lead routing. Feed 6sense or Clearbit enrichment data into routing logic so leads route based on real company attributes (firmographic + intent) rather than just form-fill data. Create a "MOps Health" dashboard in Salesforce with five widgets: lead routing accuracy %, MQL SLA breach rate, duplicate rate, missing required field rate, and lead-to-contact conversion rate. Share this dashboard in every weekly MOps standup.

**Marketo / HubSpot:**
Use Marketo's Program Performance report + Revenue Cycle Analytics (or HubSpot's attribution reports) as the baseline for measuring MOps efficiency improvements. Build a "MOps KPI" smart list that flags records with data quality issues (missing company, invalid email format, duplicate email) — review weekly and auto-trigger enrichment workflows.

**Zapier / Make (Integromat):**
Build a "MOps Nerve Center" automation in Zapier/Make that monitors key failure signals: (1) MQL created but no SDR task in Salesforce within 1 hour → Slack alert to MOps manager; (2) Campaign UTM missing utm_campaign parameter → Slack alert to campaign owner; (3) Lead enrichment API returns null for Company → trigger Clearbit enrichment. These three automations alone catch the most common MOps failures in real-time.

**Notion / Asana / Monday.com:**
Create a Campaign Operations Checklist template in your project management tool with 25-item QA checklist (UTM validation, tracking pixel check, form test, email rendering test, landing page mobile check, list suppression applied, sales notification configured, reporting dashboard linked). Use AI to auto-generate a campaign-specific checklist from the campaign brief — reduces campaign build QA time from 2 hours to 20 minutes.

**Claude / OpenAI API via Zapier:**
Connect an AI API call to your lead alert workflow: when an MQL is created, auto-generate a personalized 3-line SDR opening message using the lead's job title, company, recent Clearbit/6sense signals, and the content asset they engaged with. Load it into Salesloft/Outreach as a draft — SDR reviews and sends in <90 seconds. Reduces SDR research time per lead from 8 minutes to 90 seconds.

**Google Sheets + Looker Studio:**
Build a "MOps Efficiency Tracker" Google Sheet that aggregates weekly: campaign launch times, lead routing accuracy (sample audit), MQL SLA compliance, and data quality scan results. Connect to Looker Studio for a visual dashboard. Schedule an AI-generated weekly MOps digest every Monday morning that summarizes performance vs. targets and flags the top 3 anomalies for the MOps manager to investigate.

## Troubleshooting

**"The 90-day roadmap requires a RevOps engineer but we only have a MOps admin with no coding experience."**
Reframe the automation designs around no-code/low-code tools. Zapier, Make, HubSpot Workflows, and Marketo Smart Campaigns can handle 80% of the automations described without any coding. For the AI integrations, use Claude.ai or ChatGPT with Zapier's built-in AI steps rather than custom API calls. Adjust the timeline: Sprint 1 (quick wins) stays at 30 days; Sprints 2-3 extend to 60 days each to account for the steeper learning curve. Prioritize automations built entirely within your MAP (Marketo/HubSpot) and CRM before reaching for external automation platforms.

**"Our CRM data is so dirty that fixing routing rules doesn't help — the same bad data will just re-enter."**
Run a data quality sprint before the routing fix, not after. Use Clearbit Enrichment, ZoomInfo, or Apollo.io to bulk-enrich all active leads and contacts — this typically costs $500-2K and can clean 60-80% of your database in a week. Simultaneously, add validation rules at every entry point (web forms, CSV imports, Salesforce web-to-lead) that reject records missing Company and Work Email. Fix the input before optimizing the routing, or you'll be back to 25% misroute rate within 60 days.

**"Sales is complaining that MOps automation makes leads feel 'robotic' — they want manual personalization."**
This objection usually means the AI-generated content isn't being personalized at the account level. Upgrade the automation: instead of using only form-fill data (name, company, title), pull in real-time signals — what page they visited, what content asset triggered the MQL, the company's 6sense intent topic, and their LinkedIn headline. Feed all of this into the AI prompt. The difference between "Hi John, thanks for downloading our eBook" and "Hi John, noticed Meridian Analytics has been researching pipeline forecasting — saw you pulled our revenue prediction guide" is the difference between feeling robotic and feeling genuinely researched.
