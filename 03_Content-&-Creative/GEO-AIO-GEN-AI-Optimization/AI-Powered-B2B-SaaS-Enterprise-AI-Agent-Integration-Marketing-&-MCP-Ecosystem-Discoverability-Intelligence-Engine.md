## Troubleshooting

**Problem: AI agents are calling competitor tools instead of mine even though both are installed.**
Solution: The agent selection is driven by tool description clarity and specificity. Rewrite your tool manifest using more precise action verbs and explicit "when to use this vs. alternatives" guidance. Run A/B tests with 3 different descriptions in your staging MCP environment and measure call rates. Also ensure your tool returns structured, consistent JSON — agents learn to prefer tools that return reliable schemas.

**Problem: IT security reviews are blocking MCP tool approval, slowing enterprise adoption.**
Solution: The blocker is almost always one of three things: unclear data scope, missing audit logging, or no SSO/SCIM support. Use Section 4A output to build a security one-pager that addresses all three proactively. Add a "read-only mode" option if you don't have one — many enterprises will approve read-only access much faster than read-write. Request an explicit meeting with the CISO team rather than submitting a form; frame it as "we want to help you evaluate this safely."

**Problem: The content program (Section 3) isn't generating AI citations — our product isn't showing up when people ask about our category.**
Solution: AI citation pickup requires three elements: (1) content published on domains the LLM trusts (your site + high-authority third-party publications), (2) specific, numerical claims that differentiate your content from generic coverage, and (3) structured data markup (FAQPage, HowTo, Product schema). Audit each existing piece against these criteria. The fastest fix is usually adding specific customer outcome data with exact numbers and adding FAQ schema to your top 5 traffic pages.
