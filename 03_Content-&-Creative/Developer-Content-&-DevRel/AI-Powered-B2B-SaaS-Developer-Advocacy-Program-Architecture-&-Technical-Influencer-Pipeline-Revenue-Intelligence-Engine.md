## Troubleshooting

**Issue 1: Advocate content quality is inconsistent — some tutorials are technically excellent, others are low-quality marketing fluff.**

Fix: Implement a structured pre-publication review process: (1) Technical accuracy review by a staff engineer (30 min max), (2) Content brief compliance checklist — does it hit all required sections?, (3) "Dogfood test" — did the advocate actually run the code and verify it works? Add a published quality score to the advocate scorecard and flag advocates with < 7/10 average for a "content coaching" session. Never approve content that makes product claims the engineer reviewer can't verify.

**Issue 2: Attribution is broken — most developer conversions show as "direct" or "organic" with no advocate source, making it impossible to measure program ROI.**

Fix: Implement a three-layer dark social capture approach: (1) Community UTM discipline — every piece of advocate content must have a tagged link, use bit.ly or a custom short domain to make long UTMs shareable, (2) Sign-up survey with "How did you first hear about us?" — add "Developer tutorial/content" as a category and ask for content name, (3) IP-to-company matching (Clearbit Reveal, 6sense) on signup to correlate company-level conversions with advocate content views. Accept that 20-30% of DevRel impact will remain dark — use the survey data to apply a multiplier to measured attribution.

**Issue 3: External advocates start strong but go dormant after 2-3 content pieces — the program attrition is killing momentum.**

Fix: Diagnose before acting. Root causes split into: (a) Motivation loss — advocate doesn't see personal benefit (fix: introduce tiered rewards, public recognition, conference sponsorship opportunities), (b) Effort too high — content briefs require too much original work (fix: provide more scaffolding — outline, code scaffolds, reference architecture), (c) Product dissatisfaction — advocate actually doesn't like the product (fix: schedule quarterly product feedback calls, act on requests visibly). Add a "30-day check-in" to every new advocate onboarding — the single question that predicts churn: "What's the one thing that would make this program more valuable for you personally?"
