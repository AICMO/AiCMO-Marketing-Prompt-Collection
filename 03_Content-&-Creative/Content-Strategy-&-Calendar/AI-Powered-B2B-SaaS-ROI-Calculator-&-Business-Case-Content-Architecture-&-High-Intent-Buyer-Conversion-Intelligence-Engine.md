## Integration Tips

**Outgrow / Cuvama / Ion Interactive (calculator platforms):**
- Export calculator completion data via webhook to HubSpot or Salesforce as custom properties
- Map ROI tier (1/2/3) to contact score in MAP — add 50 points for Tier 1, 25 for Tier 2
- Trigger post-calculator email sequence via workflow automation using `calculator_completed = true` trigger

**HubSpot:**
- Create custom contact properties: `calculator_roi_pct`, `calculator_3yr_npv`, `calculator_payback_months`, `calculator_tier`
- Build enrollment trigger: Workflow → Contact → Calculator completed → Enroll in post-calculator sequence
- Sales notification: Internal email or Slack notification via HubSpot → Zapier → Slack with all calculator data points in the message body

**Salesforce:**
- Map calculator output to Opportunity fields using the following custom objects: `ROI_Calculator_Result__c` with fields for each calculated metric
- Lead routing rule: Calculator Tier 1 → Route to senior AE, Tier 2 → Route to SDR queue, Tier 3 → Enroll in long-cycle nurture
- Dashboard: Build "Calculator Pipeline Influence" report showing opportunities where `Calculator_Completed__c = true` vs. not — measure win rate and ACV delta

**Clay / Apollo / Clearbit (enrichment):**
- Trigger enrichment on email capture (before calculator results shown) to append company size, industry, tech stack, and revenue
- Use enriched company data to auto-populate "Companies like yours typically see X outcome" in the results page
- Score lead automatically using enrichment data before routing to sales

**Google Analytics 4 / Amplitude:**
- Track calculator events: `calculator_started`, `calculator_tier1_complete`, `calculator_tier2_complete`, `calculator_email_captured`, `calculator_pdf_downloaded`
- Funnel analysis: Identify which input steps have highest drop-off and A/B test copy/UX to improve
- Attribution: Tag calculator leads with UTM source to measure which channels drive highest-value calculator completions

**Notion / Confluence (internal documentation):**
- Store calculator assumptions library as a living document — update benchmarks quarterly as customer data improves
- Build sales team quick-reference: "How to use the calculator in-deal" one-pager linked in Salesforce opportunity sidebar

## Troubleshooting

**Problem: Calculator completion rate is below 20% (visitors start but abandon)**
Solution: The most common cause is asking too many inputs upfront. Restructure to show Tier 1 inputs only (3 fields max) on the first screen with immediate partial results — reveal Tier 2 inputs after showing a "teaser" result to hook the buyer. Also check mobile experience — 40–50% of initial calculator visits are mobile and most calculators are optimized for desktop only.

**Problem: Sales team isn't using calculator results in outreach — they're ignoring the handoff package**
Solution: The handoff package is being sent via email that gets lost. Integrate it directly into Salesforce as a native Activity feed item and display it prominently on the Lead/Opportunity record. Add a required "Calculator Results Reviewed" checkbox to the lead qualification workflow. Train AEs with a specific opening line template: "I saw you ran our ROI calculator and calculated [X% ROI] — I wanted to see if those numbers matched what you're seeing internally." When AEs see personal response rates improve 30–40% using calculator context, adoption follows.

**Problem: CFO isn't accepting the calculator's business case — says the numbers feel "marketing-generated"**
Solution: The CFO credibility problem comes from (a) overly optimistic improvement factors or (b) benchmarks that aren't cited. Fix by: showing all assumptions explicitly in the output document (not hiding them), including a "conservative scenario" that's clearly labeled, citing specific third-party research for benchmarks (Gartner, Forrester, industry associations), and adding a "validate with your data" section where the CFO can override any assumption and see the math update in real time. If possible, include actual customer contract metrics — "[Customer Name], a [similar company], achieved [X outcome] in [Y months]" — rather than "customers like yours."
