## Integration Tips

- **Slack/Teams**: Set up a #competitive-intel channel. Use the Phase 5 Slack template directly. Pin updated battlecards as channel bookmarks. Zapier can auto-post G2 review alerts when specific competitor names appear.
- **Highspot / Seismic**: Tag battlecard sections with signal dates so reps see freshness indicators. Use version history to track changes.
- **Salesforce**: Add a "Competitive Signal" object or tag to Opportunities. When a rep logs a competitor in a deal, trigger an automated battlecard share via Salesforce flow.
- **Google Alerts / Feedly**: Set up competitor name + ["pricing", "raises", "launches", "acquires"] as monitoring queries. Route digests to a shared inbox that feeds this prompt weekly.
- **G2 / Capterra**: Use G2's Compare API or manually export "Recent Reviews" filtered by competitor monthly. Feed the sentiment patterns into Phase 1 signal inputs.
- **Notion/Confluence**: Build a live Competitive Intel Hub page. Use this prompt output to populate sections. Embed Slack alert links for searchability.
- **Clay / Apollo**: Monitor competitor hiring patterns (engineering headcount vs. go-to-market headcount signals product vs. sales investment cycles).

## Troubleshooting

**Problem: Signals feel noisy — too many Low-threat items generating unnecessary updates**
Solution: Add a minimum Threat/Opportunity Score threshold (e.g., only auto-update battlecards for scores ≥6). For scores <6, batch into a monthly "low-priority watch" digest instead of triggering immediate response. Calibrate thresholds by reviewing which signal types have historically correlated with lost deals.

**Problem: Sales reps ignore battlecard updates because there are too many**
Solution: Limit forced updates to 1-2 per competitor per month maximum. Use the "⚡ NEW" tag sparingly — only for the single most important change. Include a "TL;DR for reps" section at the top of every update (3 bullets, plain language, what to say tomorrow). Gamify adoption by tracking reps who use competitive intel in closed-won deals.

**Problem: Competitor signals are stale by the time they reach the prompt**
Solution: Automate signal collection upstream. Use Zapier or n8n to aggregate: G2 new review alerts → Google Sheet, competitor press release RSS → Sheet, LinkedIn company page post alerts → Sheet. Run this prompt weekly against the auto-populated Sheet rather than manually curating signals. For high-stakes accounts, run daily.
