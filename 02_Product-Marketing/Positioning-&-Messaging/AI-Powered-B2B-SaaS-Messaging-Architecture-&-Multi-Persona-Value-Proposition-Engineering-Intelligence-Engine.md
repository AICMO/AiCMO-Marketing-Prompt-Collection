Use it when launching a product, repositioning against competitors, onboarding a new PMM, or when pipeline stalls because "our message isn't landing."

## Integration Tips

- **Salesforce / HubSpot:** Load the persona messaging matrix into your CRM as "Persona Playbooks" — attach to lead/contact records so reps pull the right message for each stakeholder role automatically
- **Gong / Chorus:** Use the 15 discovery questions as a call scorecards checklist; track how often reps deploy objection-handling scripts vs. improvising
- **Notion / Confluence:** Create a "Message House" page for each ICP segment using Module 2 output as the template; gate with edit permissions so messaging doesn't drift across teams
- **Outreach / Salesloft:** Load Module 5's LinkedIn and cold email copy directly into sequence templates with persona tagging so reps send the right message to the right role
- **Google Optimize / VWO:** Run Module 7 A/B test hypotheses as controlled experiments on homepage; set 95% statistical significance threshold before calling winners

## Troubleshooting

**Issue: Output messaging feels generic or could apply to any competitor**
- Solution: Add 5+ customer quotes verbatim from recent win/loss interviews to the prompt. Generic messaging = generic inputs. The prompt needs your customers' actual language, not your internal assumption of their language. If you don't have verbatim quotes, run the Voice of Customer prompt first before executing this one.

**Issue: Sales team ignores the messaging framework and keeps improvising**
- Solution: The adoption barrier is usually format, not content. Take the objection-handling matrix and load it into your call recording tool (Gong/Chorus) as scorecards. Reps adopt tools that live in their workflow — not documents they have to open separately. Also: run a 30-minute "message bootcamp" where reps roleplay using the scripts on real objections before launch.

**Issue: Economic buyer and technical buyer messaging sounds identical**
- Solution: Re-run Module 2 with explicit JTBD statements grounded in each persona's quarterly performance metrics. The CFO cares about forecast accuracy and capital efficiency. The VP Eng cares about integration complexity and uptime SLAs. If your bullets still sound the same, you're writing features, not outcomes. Rewrite every bullet starting with a concrete business result ("Reduce quarter-end forecast scramble") before adding the mechanism.
