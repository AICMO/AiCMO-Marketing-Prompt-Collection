## Overview
Designs and executes a systematic positioning validation system that tests multiple messaging variants across micro-experiments before committing to campaigns, using AI to generate positioning hypotheses, design multi-channel test matrices, interpret performance signals, and identify the winning message architecture that maximizes pipeline conversion. Use this when you have competing positioning options and need data (not opinions) to decide, when your current messaging feels stale but you're unsure which direction to move, or when a new product launch requires validation before a full GTM investment.

## Integration Tips

**HubSpot:**
- Create a custom contact property "first_message_variant_engaged" to track which positioning variant a contact first converted on — use this for downstream attribution
- Build a workflow that auto-tags contacts based on which landing page URL they converted on (each variant gets a unique /lp/ URL with UTM)
- Use HubSpot A/B email testing for simultaneous subject line variants — minimum 1,000 contacts per variant for reliable data

**Salesforce:**
- Add a "Positioning Influence" field to Opportunity to track which message variant the champion was exposed to during evaluation — enables win rate analysis by message
- Build a report: Win Rate by Positioning Variant Last Touch — requires SDR/AE discipline in logging outreach variant used in activity records
- Create a Campaign hierarchy: Parent "Q3 Positioning Test" → Child campaigns per variant — enables pipeline attribution to each message variant

**LinkedIn Campaign Manager:**
- Use LinkedIn's built-in A/B test feature for Sponsored Content (available in Campaign Groups) — do NOT manually split — LinkedIn's native test ensures equal audience distribution
- Set frequency cap at 4 impressions/week per user — prevents ad fatigue from inflating CTR decay
- Use LinkedIn Insight Tag conversion tracking mapped to your demo scheduling page (Calendly / Chili Piper) to capture demo conversion by ad variant

**Outreach / Salesloft:**
- Create parallel sequence variants: Sequence A (Hypothesis 1 messaging) vs. Sequence B (Hypothesis 2 messaging) — assign reps to accounts randomly using territory rotation
- Use Outreach's template analytics to compare "reply rate" and "positive reply rate" by sequence at 200+ send threshold
- Governance rule: do not let reps choose which sequence to use — must be randomly assigned or variant signal is contaminated

**Google Optimize / AB Tasty:**
- Run homepage headline test with minimum 30% traffic share per variant
- Use GA4 goals for "demo page reached" as primary conversion goal
- Do NOT use micro-conversions (scroll, time on page) as primary success metric — they don't predict pipeline

**Notion / Confluence:**
- Build a "Positioning Test Registry" with one row per variant: ID, hypothesis, test start date, channels, primary metric result, statistical significance, MMF score, and status (testing / scaled / retired)
- Require all messaging changes across the company to reference a Positioning Test Registry row — creates institutional memory and prevents regression to old messaging

## Troubleshooting

**Problem: Every variant performs similarly — no clear winner after 4 weeks**
- Root cause 1: Variants are not meaningfully differentiated — all express similar emotional hooks or category frames in slightly different words
- Fix: Run "extreme hypothesis" test — deliberately test your most conservative, safe message vs. your most provocative, category-disrupting message. Incremental variants will never reveal strong signal; contrast reveals signal.
- Root cause 2: Audience quality is too mixed — testing on broad LinkedIn audiences includes too many non-ICP viewers who dilute signal
- Fix: Retarget to your existing contact database (Matched Audiences in LinkedIn Campaign Manager) or to the specific job title + company size combination of your ICP — smaller audience, cleaner signal

**Problem: A message wins on LinkedIn but fails completely in cold email**
- Diagnosis: This is channel-specific language, not category-level positioning. LinkedIn rewards bold, industry-perspective hooks ("The ops backlog era is over"). Email rewards specific, personal hooks ("I noticed Verdant tracks X — here's what our VP Ops customers do differently").
- Fix: Do not use this message as your core positioning — extract the emotional theme and translate it into channel-appropriate language. Positioning is the idea; execution varies by channel.
- Red flag to avoid: Declaring a positioning winner based on email reply rate alone. Email rewards relevance to the individual recipient more than it rewards category-level positioning resonance.

**Problem: Sales team keeps reverting to old messaging despite test results proving a winner**
- Root cause: Reps don't trust data they didn't generate and weren't part of designing
- Fix: Before running the test, involve 2-3 top-performing SDRs and AEs in hypothesis generation. Humans defend what they helped create. Brief the team on WHY the winner won (the data, the mechanism, the buyer psychology) — not just WHAT won. Hold a 30-minute "positioning debrief" where reps can ask questions and see the actual test data firsthand. Then document the winning message in the sales playbook as a non-negotiable standard rather than a suggestion.
