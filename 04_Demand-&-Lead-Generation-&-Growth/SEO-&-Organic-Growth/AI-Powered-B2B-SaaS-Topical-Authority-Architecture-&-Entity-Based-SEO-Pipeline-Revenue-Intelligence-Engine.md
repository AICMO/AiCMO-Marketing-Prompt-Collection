## Integration Tips

- **Semrush / Ahrefs**: Export topic cluster ranking data weekly; feed into Agent 1 (Topical Gap Monitor) via CSV or API. Use Semrush's Topical Authority score as a benchmark proxy.
- **Google Search Console**: Connect to Looker Studio for cluster-level click/impression tracking. Create custom dimensions tagging pages by topic pillar to see pipeline contribution per cluster.
- **HubSpot / Salesforce**: Add a custom property "Organic Topic Cluster" to Contacts and Opportunities. Set via workflow triggered when first-touch source = organic and page URL contains cluster-specific URL patterns.
- **Notion / Confluence**: Use Agent 3 output (content briefs) directly as Notion database entries. Add fields for: entity target, cluster assignment, pillar link, schema type, and publication status for full editorial pipeline visibility.
- **Clearscope / MarketMuse**: Run each content brief output through Clearscope for entity optimization scoring before production. Target 50+ grade for entity completeness before publishing any pillar or cluster piece.
- **Zapier**: Automate the pipeline: New Semrush rank drop detected → Slack alert to SEO lead → Auto-create Notion brief task with gap context → Assign to content writer.
- **Screaming Frog**: Schedule monthly crawl to audit internal linking health. Feed crawl data into the topical architecture map to ensure every cluster piece links back to its pillar and no orphan pages exist.

## Troubleshooting

**Problem: AI output produces generic topic clusters with no real buyer specificity (e.g., "what is financial planning" instead of "SaaS CFO financial planning").**
Solution: Add explicit ICP constraints to the prompt: "Every topic must be relevant to [ICP_JOB_TITLE] at a [COMPANY_SIZE] [COMPANY_TYPE] company. Reject any topic that a consumer or non-SaaS buyer would search for." Also add your ICP's specific vocabulary in the prompt — terms they use in sales calls, Gong recordings, or win/loss interviews.

**Problem: Entity associations the AI recommends are too abstract and don't translate into rankable content.**
Solution: After generating entity associations, run a second prompt: "For each entity association listed, generate a specific content asset title and target keyword that establishes this entity relationship and could rank in the top 10." This forces the AI to translate entity theory into executable content pieces with real search demand.

**Problem: Pipeline attribution is impossible because organic traffic doesn't appear in CRM or self-reported attribution is unreliable.**
Solution: Implement a two-layer attribution system: (1) Technical: UTM parameters on all internal CTAs (demo buttons, contact forms) tagged with topic cluster (e.g., utm_content=cluster_saas-scenario-planning); (2) Self-reported: Add a "Where did you first learn about us?" free-text field to all demo request forms and train your SDRs to ask this question on every discovery call. Aggregate monthly and map free-text responses to topic clusters.
