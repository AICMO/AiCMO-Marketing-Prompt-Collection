## Integration Tips

- **Amplitude/Mixpanel:** Create a "Viral Funnel" chart with events: `invitation_trigger_shown` → `invitation_sent` → `invitation_opened` → `invited_user_signup` → `invited_user_activated`. Set up a K-factor formula property using the User Composition chart.
- **Segment:** Add a `viral_origin` trait to user profiles at signup if `utm_source=invitation` or referral cookie is present. Sync this trait to Salesforce as a Contact field.
- **Salesforce:** Create a custom Account field `Viral_Origin__c` (checkbox) and `Viral_Inviter_Account__c` (Account lookup). Build a report comparing ACV, win rate, and churn rate for Viral vs. Non-Viral origin accounts quarterly.
- **HubSpot:** Use the Contact property `Original Source Drill-Down 1` to tag viral-origin contacts. Create a Workflow that tags any contact with `utm_medium=viral_invite` and maps them to a Viral Origin list for cohort reporting.
- **Looker/Tableau:** Build a K-factor time-series dashboard showing daily K calculation (invitations sent ÷ MAU × acceptance rate), viral cycle time trend, and % of new signups from viral sources. Alert when K drops below 0.3 week-over-week.
- **Notion:** Use the Advanced Version output to populate a "Viral Loop Analytics" page with sections mirroring each deliverable. Link directly to the relevant Amplitude/Mixpanel dashboard from each metric row.
- **Zapier:** Set up a Zap: when a new Salesforce Account is created with `Viral_Origin__c = True`, add to a Google Sheet "Viral Cohort Tracker" and send a Slack notification to the growth channel with account name, inviter account, and signup date.

## Troubleshooting

**Problem: Can't calculate K-factor because invitations aren't tracked in product analytics**
Solution: Add three tracking events immediately: `invitation_modal_opened` (triggered when user sees the invite UI), `invitation_sent` (triggered on submission with `{invitee_email, inviter_user_id, context}`), and `invited_user_signup` (triggered at new user registration with `{referral_code, inviter_id}`). Even 2 weeks of data is sufficient to establish a meaningful K-factor baseline. Until then, approximate K using email delivery logs if invitations are sent via email.

**Problem: Viral-origin attribution is contaminated because most users also saw paid ads**
Solution: Use a "first viral touch wins" attribution model for identifying viral-origin accounts — if a user clicked an invitation link at any point before signup, classify them as viral-origin regardless of subsequent ad exposure. This is the most conservative and investor-credible approach. For marketing mix modeling, treat viral as a separate channel with zero media spend but positive engineering/design cost.

**Problem: K-factor looks strong (>0.7) but viral signups aren't converting to paid at expected rates**
Solution: The invitation mechanic is probably attracting passive viewers (invited to view output) rather than active collaborators (who would need paid features). Audit invited-user activation rates: what % of invited users perform a "creator action" (recording, annotating, editing) vs. a "viewer action" (commenting, watching)? If <25% of invited users perform creator actions within 14 days, redesign the invitation flow to target invitees who will need paid features, not free-tier viewers.
