Use this when CPL looks healthy but sales is complaining about lead quality, when you suspect audience saturation, or when you need to rebuild targeting from a revenue-first perspective.

## Integration Tips

- **LinkedIn Campaign Manager:** Export audience performance from "Reporting" → "Demographics" tab to get seniority, job function, company size breakdowns — paste directly into the Advanced Version input table
- **Google Ads:** Use "Audience Manager" → "Audience Insights" + "Search Terms" report together to separate intent quality from audience quality signals
- **HubSpot:** Build a custom report: "Contacts by Lead Source + Latest MQL Date + Deal Stage" to calculate MQL→SQL rates by paid channel segment; export as CSV for this prompt
- **Salesforce:** Use "Campaign Influence" report filtered by campaign type = "Paid" to pull pipeline and closed-won revenue per audience segment; combine with CRM campaign UTM parameters
- **Bombora / G2 Intent:** Feed intent topic scores into the "Audience Saturation" section to identify which ICP accounts are in-market right now vs cooling off — this changes bid multiplier recommendations
- **Looker / Tableau:** Create a segment-level dashboard with ICP Fit Score as the primary axis; automate weekly refreshes so targeting decisions are always based on 30-day rolling data, not stale 90-day snapshots
- **Zapier / Make:** Build an automation that fires when a segment's MQL→SQL rate drops below 20% for 14 consecutive days — auto-generates a Slack alert with this prompt pre-loaded for immediate analysis

## Troubleshooting

**Problem:** My audience segments are too broad and I don't have performance data broken down by targeting criteria.
**Solution:** In LinkedIn Campaign Manager, go to Reporting → Demographics for any active campaign and download the performance breakdown by "Job Seniority," "Job Function," and "Company Size" as separate CSVs. For Google, use the Audience segment report under "Campaigns → Audiences → View Table." These demographic overlays give you the targeting-level data even if you didn't build separate campaigns per segment.

**Problem:** My MQL→SQL data doesn't tie back to specific ad audiences because attribution is broken in my CRM.
**Solution:** Use UTM parameters as a proxy: set up unique UTMs per audience (e.g., utm_audience=linkedin_construction_vp) and filter your CRM contact list by UTM source + medium + campaign. Even if multi-touch attribution is missing, this first-touch audience signal is enough to calculate directional SQL rates per targeting combination. Start with this now, then invest in proper attribution infrastructure using the Attribution Engine prompt.

**Problem:** My audience sizes are too small for LinkedIn to generate statistically significant data (fewer than 50 SQLs per segment in 90 days).
**Solution:** Collapse adjacent segments for analysis: merge "Director + VP" into one seniority tier, and "Construction + Civil Engineering + Real Estate" into one industry tier. Run the ICP Fit Score analysis on merged segments. Once you identify the winning combination, break it back into sub-segments for future testing. Statistically, you need at least 30 SQLs per segment to draw meaningful conclusions — work backward from that to set minimum campaign duration before making pause/kill decisions.
