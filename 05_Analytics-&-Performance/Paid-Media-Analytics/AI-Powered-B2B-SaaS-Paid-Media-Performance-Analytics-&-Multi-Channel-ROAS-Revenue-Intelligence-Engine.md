## Integration Tips

- **HubSpot:** Connect Google Ads and LinkedIn Ads natively via HubSpot's paid media integration. Run this prompt against the "Ads Performance" report exported as CSV. Use HubSpot's attribution reporting to pull first-touch vs. last-touch pipeline data per channel.
- **Salesforce + Pardot/Marketing Cloud Account Engagement:** Export Campaign Influence report filtered by "Paid" campaign type. Cross-reference with Opportunity Source to get pipeline sourced vs. influenced split per channel.
- **Google Ads + Looker Studio:** Build a Looker Studio dashboard with Google Ads connector + BigQuery CRM data. Feed the dashboard export into this prompt monthly.
- **Northbeam / Triple Whale / Rockerbox:** If using an MTA platform, export the channel-level revenue attribution report and use it as the primary data source — it handles the multi-touch math already.
- **Zapier/Make automation:** Set up a weekly Zap that pulls Google Sheets paid media report + CRM pipeline data and sends the merged CSV to a Claude API call running this prompt, then posts output to a Slack channel for Monday morning review.
- **RevOps / CRM sync:** Ensure all paid channels pass UTM parameters consistently (utm_source, utm_medium, utm_campaign, utm_content) into your CRM to make the pipeline attribution data reliable before running this analysis.

## Troubleshooting

**Problem: Pipeline data doesn't tie back cleanly to paid channels — attribution gaps everywhere.**
Solution: Start with self-reported attribution. Add "How did you hear about us?" as a required field on all form fills. Cross-reference with UTM data. Where they conflict, weight self-reported 60% / UTM 40% for directional accuracy. Flag to ops team to fix UTM governance before the next cycle.

**Problem: The AI output recommends scaling LinkedIn but finance won't approve the budget increase without proof of incrementality.**
Solution: Run a 4-week LinkedIn holdout test: pause LinkedIn ads for 20% of your ICP target accounts (use a random sample from your TAL). Compare pipeline velocity for the holdout group vs. exposed group. Use the incrementality result as the CFO-ready proof point.

**Problem: Data from different channels is in different date ranges or uses different conversion definitions.**
Solution: Normalize everything to a single 30-day window (last calendar month) and use a single conversion definition — "Marketing Qualified Lead" as defined in your CRM — before running this prompt. Mixing CPL definitions across channels (Google's form fill vs. LinkedIn's lead gen form vs. Meta's lead ad) creates false comparisons that lead to wrong reallocation decisions.
