## Overview
Transforms raw website analytics data into a revenue-focused performance intelligence report that identifies exactly which pages, traffic sources, and visitor behaviors are driving (or killing) pipeline. Use this when website traffic isn't converting to demos/trials, when you need to justify web investment to the CFO, or when you need a data-driven roadmap to improve your site's contribution to pipeline without adding headcount.

## Integration Tips

- **GA4 + BigQuery:** Export raw GA4 event data to BigQuery for full session-level analysis. Use the `events_*` table with `event_name = 'page_view'` and `event_name = 'generate_lead'` to build your conversion funnel query. Join with `session_traffic_source_last_click` to get accurate source attribution. Run this query weekly and push results to a Looker Studio dashboard for the CMO.
- **HubSpot:** Use the "Original Source" and "Original Source Drill-Down 1/2" contact properties to correlate website traffic sources with actual closed-won revenue (not just demo requests). Build a HubSpot custom report: Source → Contacts → Deals → Closed Revenue. This gives you true pipeline-per-source-per-session math, not estimated pipeline.
- **Hotjar / FullStory:** Set up Hotjar "Funnels" to track the exact micro-stage dropoff defined in this prompt. Watch session recordings filtered to "Pricing page visitors who did NOT convert" — in the first 10 recordings you will identify the specific UX issue causing abandonment. Tag recordings with "pricing_exit" to build a library of evidence for the engineering team.
- **6sense / RB2B / Clearbit Reveal:** Connect your de-anonymization tool to GA4 via custom dimensions. When a known company is identified, fire a custom event (`company_identified`) and set the company name, industry, and employee count as event parameters. This lets you segment GA4 reports by ICP firmographic data — the most powerful version of web analytics for B2B.
- **Salesforce:** Create a custom field "First Web Touchpoint" on the Lead object, populated by the UTM parameters from the first website session. After 6 months, run a Salesforce report correlating First Web Touchpoint → Closed-Won revenue. This is your true website attribution model, accounting for full sales cycle lag.
- **Google PageSpeed Insights API:** Automate Core Web Vitals monitoring. Use the PageSpeed Insights API to run weekly checks on your top 10 most-visited pages and send alerts to Slack (#web-performance) if any metric degrades. Set thresholds: LCP >3s = yellow alert, LCP >4s = red alert with auto-ticket creation in Jira.
- **Zapier / Make:** When a high-intent behavioral signal occurs (pricing page visited 2+ times in a session, demo page visited but form not submitted, case study downloaded), trigger a real-time Slack notification to the SDR team with the visitor's company name (from 6sense), pages visited, and time on site. This converts anonymous web intelligence into same-day outbound action.

## Troubleshooting

**Problem: GA4 conversion data doesn't match HubSpot form submissions — the numbers are off by 30-40% and the CMO doesn't trust either source.**
Solution: This is the most common B2B web analytics problem and it's caused by GA4 events firing before form submissions complete (especially on multi-step forms or SPAs), ad-blockers suppressing GA4 tracking for 15-25% of privacy-conscious B2B buyers, and bot traffic inflating GA4 sessions. Fix: Use HubSpot as your primary conversion source of truth (it records form submissions server-side), and use GA4 as your traffic and behavioral data source. Build a reconciliation model: HubSpot conversions ÷ GA4 sessions = your "true" conversion rate. Document the methodology and present it to the CMO once so it's settled.

**Problem: Core Web Vitals are in the "Poor" range but engineering won't prioritize the fix because they don't see a direct revenue impact.**
Solution: Quantify it in dollars using this exact formula and present it as a business case in their sprint planning: (Monthly sessions from SEO) × (Organic ranking suppression from poor CWV — estimate 5-15% traffic reduction based on Google's ranking factor research) × (Current organic CVR) × (ACV) × (Close rate) = Monthly revenue cost of technical debt. For a site doing $200K/month in organic pipeline, a 10% traffic suppression from poor CWV = $20K/month in lost pipeline = $240K/year. Frame it as: "We are choosing to lose $240K/year to avoid a 2-sprint engineering investment."

**Problem: Traffic is growing month-over-month but pipeline from the website is flat — the CMO wants to know if the website is broken or if it's a traffic quality issue.**
Solution: Run the traffic quality diagnostic: segment conversion rate by source and by new-vs-returning visitors separately. If overall CVR is declining while organic CVR is stable, the problem is traffic mix shift — a growing proportion of low-quality paid or social traffic is diluting the average. The website isn't broken; the traffic mix changed. Present as: "Organic conversion rate held steady at 4.8%. Overall CVR declined from 3.9% to 2.8% because LinkedIn Ads (1.4% CVR) now represents 23% of sessions vs. 11% last quarter. Solution: optimize LinkedIn landing pages or reduce LinkedIn spend until we fix the audience targeting."
