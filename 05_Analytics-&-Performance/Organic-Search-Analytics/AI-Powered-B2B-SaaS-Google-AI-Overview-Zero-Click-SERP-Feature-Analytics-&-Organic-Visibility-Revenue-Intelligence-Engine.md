## Troubleshooting

**Problem: Google Search Console shows AI Overview impressions but doesn't differentiate them from standard results.**
Solution: Google consolidates AI Overview impressions into the standard "Queries" report, counting them at an equivalent position of ~1. To isolate AI Overview impact, export CTR data for all queries ranked position 1–3 and look for queries where CTR is <3% at position 1 (well below typical 20–30% CTR at #1 for standard results). These are strong candidates for AI Overview displacement. Cross-reference with a tool like SE Ranking, BrightEdge, or manual SERP snapshots to confirm AI Overview presence.

**Problem: AI Overview appearances are inconsistent — the same query shows AI Overview on some searches but not others.**
Solution: Google AI Overviews trigger probabilistically based on query type, user context, search history, and device. For consistent measurement, use automated SERP monitoring tools (Authoritas, BrightEdge, SE Ranking) that run queries from standardized environments at set intervals. Take 10+ snapshots of the same query across different times and user contexts to establish a reliable trigger rate. Report on AI Overview inclusion as a "trigger rate" (e.g., "AI Overview triggered 7 of 10 times for this query") rather than a binary metric.

**Problem: The zero-click pipeline attribution model requires brand tracking survey data we don't have.**
Solution: If brand tracking surveys are unavailable, use branded keyword search volume in GSC as a proxy for unaided awareness. Set up a weekly export of (a) total non-branded impressions from zero-click SERP features and (b) branded query volume for the same week, then run a 6-week lagged correlation analysis quarterly. As a starting point, use the industry benchmark that every 10,000 non-click branded impressions correlates with approximately 3–5% uplift in brand search volume among the exposed audience segment — calibrate this ratio using your own historical data over 2–3 quarters.
