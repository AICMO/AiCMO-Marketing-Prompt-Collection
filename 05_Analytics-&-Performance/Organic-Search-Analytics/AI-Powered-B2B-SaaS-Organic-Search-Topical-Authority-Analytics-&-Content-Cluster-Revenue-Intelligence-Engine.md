### MODULE 1: TOPICAL AUTHORITY BASELINE MAPPING

For each cluster provided, produce:

**Authority Coverage Score (ACS):**
- Enumerate the expected subtopics for this cluster (e.g., for "customer health scoring": what is a health score, how to build one, leading vs lagging indicators, scoring models, tools comparison, implementation guide, integration with CSM workflow, benchmarks by industry)
- Count how many subtopics WE have content for vs each competitor
- Calculate: ACS = (subtopics covered / total expected subtopics) × 100
- Flag: RED if ACS < 50%, YELLOW if 50-75%, GREEN if >75%

**Depth Quality Index (DQI):**
For pages we DO have, evaluate:
- Content comprehensiveness (does it fully answer the query? 1-5)
- Semantic richness (does it use expert vocabulary, examples, data? 1-5)
- Internal link density to related cluster pages (1-5)
- Backlink authority of the page vs competing pages (relative)
- DQI = average of above scores

**Competitive Authority Position:**
For each cluster, classify our position:
- AUTHORITY LEADER: We rank #1-3 for >60% of cluster keywords
- AUTHORITY CONTENDER: We rank #1-10 for 30-60% of cluster keywords
- AUTHORITY CHALLENGER: We rank #1-10 for <30% of cluster keywords

### MODULE 3: CONTENT CANNIBALIZATION FORENSICS

**Cannibalization Detection Protocol:**

Step 1 — Intent Overlap Scan:
For each cluster, identify all pages that target the same primary search intent. Flag any cluster where 2+ pages compete for the same keyword group (search volume overlap >40%).

Step 2 — Position Fluctuation Analysis:
Cross-reference with GSC position data. If a keyword shows >5 position swings over 30 days, likely multiple pages are triggering Google to alternate between them.

Step 3 — Link Equity Dilution:
Map internal links. If backlink equity is split across 3+ similar pages, consolidation will concentrate authority.

**Cannibalization Fix Playbook:**
For each identified cannibalization cluster:
1. CONSOLIDATE: Pick the strongest page (highest DA links + conversion rate), redirect others to it, merge best content
2. DIFFERENTIATE: If pages serve genuinely different intents, clarify each page's unique angle and add canonical signals
3. DELETE-AND-REDIRECT: Remove thin duplicate pages entirely if they generate zero organic traffic or conversions

Expected outcome per consolidation action: Model 15-30% ranking improvement for the surviving URL within 45-60 days.

## Integration Tips

- **Semrush/Ahrefs:** Export "Keyword Gap" report for each competitor pair. Use "Topic Research" to enumerate expected subtopics per cluster. Feed this data directly into Module 1's coverage scoring.
- **Google Search Console:** Export page-level performance for the past 12 months. Use the query filter to segment by cluster keywords. Connect to Looker Studio for automated authority trend tracking.
- **HubSpot/Salesforce:** Create a UTM source field that captures organic cluster attribution. Map CRM pipeline to organic sessions by landing page to validate the revenue impact projections.
- **Notion/Confluence:** Use Module 4's roadmap template as your content brief system. Link each brief directly to the Ahrefs/GSC data snapshot that justified the investment.
- **Screaming Frog or Sitebulb:** Run an internal link audit to identify pages in each cluster that lack incoming links. Module 3's cannibalization fix requires accurate link equity mapping.
- **Zapier/Make:** Set up automated Search Console → Google Sheets → Slack alerts when any priority cluster page drops 5+ positions. Enables weekly authority monitoring without manual checks.

## Troubleshooting

**Problem:** AI produces generic subtopic lists that don't reflect B2B SaaS buyer intent.
**Solution:** In the quick version, add a constraint: "All subtopics must be queries a [your ICP title] would search when evaluating software — not general educational content. Mark each as awareness, consideration, or decision stage." This forces practitioner-level specificity.

**Problem:** Revenue projections seem too optimistic — gap between modeled and actual pipeline from organic.
**Solution:** Calibrate your session-to-MQL conversion rate separately for organic traffic vs all channels (organic typically converts 30-60% better than paid). Check if your CRM attribution is capturing organic-first touches that close via other channels later. Apply a 0.6-0.8 confidence multiplier to first-pass projections until you have 90 days of actual data to validate.

**Problem:** Competitor ACS comes out much higher but their actual rankings aren't proportionally better.
**Solution:** Topical breadth (coverage score) is only one authority signal. Cross-reference with domain-level authority (DR/DA) and page-level backlink quality. A competitor with 80% ACS but DR 45 may lose to a competitor with 60% ACS but DR 70 — particularly in competitive SERP environments above position 5.
