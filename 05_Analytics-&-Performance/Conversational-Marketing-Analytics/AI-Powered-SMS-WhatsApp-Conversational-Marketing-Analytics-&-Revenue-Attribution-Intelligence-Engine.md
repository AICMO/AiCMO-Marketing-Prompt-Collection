## Integration Tips

- **Klaviyo**: Use the Analytics → Revenue tab filtered by channel = SMS; pull flow reports via Reporting → Flows → filter by SMS message type; export campaign performance CSV for RPM calculations
- **Attentive**: Export from Analytics → Campaigns dashboard; use the Attribution Report to compare last-touch vs. assisted; Subscriber Growth report for list health metrics
- **Postscript**: Pull from the Analytics tab — Campaigns, Automations, and Subscribers sections separately; Postscript natively shows RPM and opt-out rate per campaign
- **HubSpot SMS**: Use the Marketing → SMS section; connect to Contact Lifecycle Stage data for segment-level analysis; use Custom Reports for cross-channel attribution
- **Twilio**: Requires custom Looker/Data Studio dashboard — pull delivery, click, and opt-out events via Twilio Messaging API; join with CRM revenue data for RPM
- **WhatsApp Business API (Meta)**: Monitor in Meta Business Manager → WhatsApp Manager → Insights; template quality ratings visible in WhatsApp Manager → Message Templates
- **Cross-channel attribution**: Use Northbeam, Triple Whale, or Rockerbox for unified DTC attribution across SMS, email, and paid; configure SMS as a distinct channel with proper UTM parameters on all SMS links
- **Data pipeline**: Pipe Klaviyo or Attentive event data into BigQuery or Snowflake via their native connectors; join with Shopify/Stripe order data for true cohort-level LTV analysis

## Troubleshooting

**Problem:** SMS revenue looks artificially high because Klaviyo's 5-day attribution window overcredits SMS when email campaigns ran in the same window.
**Solution:** Run a holdout test — suppress 10% of your list from SMS campaigns for one month and compare purchase rates vs. exposed group. Use this to calculate true incrementality and apply a discount factor to attributed revenue. Alternatively, shift to a 1-day click window for SMS to reduce overlap.

**Problem:** Opt-out rate is high but it's unclear which segment or message type is causing it.
**Solution:** Export campaign-level opt-out data and cross-reference the segment targeted and message category (promotional vs. urgency vs. educational). Most platforms allow opt-out segmentation by campaign; if not, build a suppression list of opt-outs and look at their last received message in your CRM to identify patterns.

**Problem:** WhatsApp marketing templates keep getting rejected by Meta or the block rate is creeping up.
**Solution:** Review rejected templates against Meta's commerce policy — specific language around "free," "limited time," and external links triggers flags. For block rate, audit your recent template copy for urgency language and frequency; reduce sends by 30% to the highest-engagement segment first, then gradually reintroduce after the block rate drops below 0.2%. Monitor template quality scores in WhatsApp Manager weekly.
