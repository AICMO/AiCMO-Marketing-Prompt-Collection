## Overview
Transforms raw keyword ranking data from Google Search Console, Semrush, or Ahrefs into a revenue-attributed keyword intelligence system — tracking ranking movements, identifying decay before it hits pipeline, attributing ARR to keyword clusters, and automating competitive position monitoring. Use this when you need to prove the pipeline value of organic search to leadership, identify which keyword positions are worth defending versus investing in, or build a continuous ranking intelligence operation that surfaces revenue-relevant signals weekly without manual analysis.

- **Content Freshness Decay**: Page hasn't been updated in >12 months while competitors updated recently. Diagnosis: check last-modified date and compare content depth to ranking competitors.

## Integration Tips

- **Google Search Console API**: Export the full Performance report (queries, pages, countries, devices) on a rolling 90-day basis programmatically. Use the `dimensionFilterGroups` parameter to separate branded vs. non-branded queries automatically — this is the most important segmentation for accurate organic revenue attribution.
- **Semrush Position Tracking API or Ahrefs Rank Tracker API**: Both offer daily position tracking with historical data. Set up a keyword group for each commercial cluster (evaluation-stage, competitor, category) and pull via API into a BigQuery table. Calculate 7-day, 30-day, and 90-day position delta automatically.
- **HubSpot or Salesforce CRM**: Create a custom field "Original Organic Keyword Theme" (populated via UTM parameters or self-reported survey on first contact form). Run quarterly analysis to identify which keyword themes source the highest ACV deals and adjust keyword investment accordingly.
- **Looker Studio**: Build a live Keyword Revenue Dashboard with GSC and CRM connectors — show keyword cluster → clicks → MQLs → pipeline → closed ARR in a single funnel view. Add competitor position overlay using Semrush API. Refresh daily. Share as a public link with CMO and board.
- **Zapier / Make.com**: Connect Semrush webhook (position change alert) → filter for keywords with estimated ARR >$50K → create a Notion or Jira card with revenue context pre-populated → Slack notification to SEO channel. Zero human monitoring required for high-value keyword movements.
- **Google Sheets + Apps Script**: For teams without Semrush/Ahrefs API access, use Google Sheets Apps Script to pull GSC data weekly, calculate position delta vs. prior week, and email a ranked list of position movers sorted by estimated revenue impact. Entirely free to build and run.

## Troubleshooting

**Problem:** GSC shows aggregate "average position" data but many keywords have low impression counts that make position data unreliable (GSC samples data for low-volume queries).
**Solution:** Filter your GSC export to only include keywords with ≥100 impressions per month before building the revenue model. For lower-volume keywords, use Semrush or Ahrefs exact position tracking which pulls from their own crawler rather than the sampled GSC data. Accept that the revenue model is directionally accurate, not precise — communicate this to leadership and emphasize the trend lines rather than exact ARR figures. Use error ranges: "This keyword cluster is worth $200K–$400K ARR annually depending on conversion rate assumptions."

**Problem:** Organic pipeline attribution is wildly underreported in the CRM because most B2B buyers research through organic search months before they ever fill out a form or identify themselves.
**Solution:** Layer self-reported attribution questions into your demo request and trial signup forms: "How did you first hear about us?" and "What were you searching when you found us?" Analyze the responses monthly to build a keyword-to-deal map that supplements CRM tracking. Also implement Google Analytics 4 with longer attribution windows (90-day lookback) and cross-reference with GSC position data to identify high-impression keywords that are clearly influencing pipeline even when not captured as "first touch" in CRM.

**Problem:** Content team pushes back on keyword-based content briefs, arguing that "writing for search engines" produces worse content than writing for humans.
**Solution:** Reframe the conversation entirely. The revenue model output from this prompt is your tool: "The 340 tracked keywords in our portfolio represent $4.2M in annual pipeline. The top 10 page 2 keywords represent $620K in pipeline we're not getting because we have no or thin content on those topics. This isn't about writing for search engines — it's about making sure buyers who are actively searching for what we sell can actually find us." Show them the decay alert on "procurement software" (from the example): "We lost $33,600 ARR this quarter because a competitor published a better page on this topic. Content quality is exactly what this is about."
