## Integration Tips

- **Tackle.io / Suger / Clazar:** These marketplace operations platforms push transaction data directly to Salesforce via managed packages — ensure the `Marketplace_Transaction_ID`, `Hyperscaler_Platform`, and `Private_Offer_ID` fields are mapped and auto-populated on opportunity creation. Schedule daily sync jobs, not just on-close syncs.
- **Salesforce:** Create a custom Marketplace Analytics object with lookup to Opportunity to store listing impression data, ACE opportunity IDs, and EDP burndown calculations separately from core opportunity fields — prevents field bloat on the Opportunity object and enables cleaner reporting.
- **AWS ACE / Partner Central API:** Use the ACE API to pull opportunity acceptance timestamps, PDM assignment, and stage change history directly into your data warehouse — this enables PDM contribution scoring without manual CRM updates from the co-sell team.
- **Looker / Tableau:** Build the hyperscaler QBR dashboard as a white-labeled embed you can share directly with PDMs via a read-only link — this signals data sophistication and earns co-sell investment because PDMs can see their own impact in real-time.
- **Snowflake / BigQuery:** Create a `marketplace_events` table that joins AWS Seller Central API exports + CRM opportunity data + customer cloud consumption data (via AWS Cost Explorer API if customers share access) — enables the EDP burndown propensity model at scale.
- **Slack / Teams:** Automate the weekly marketplace GMV alert and EDP expiration triggers as Slack messages to the alliances Slack channel — eliminates the need for a standing weekly review and keeps the team action-oriented.

## Troubleshooting

**Problem:** Co-sell win rate appears dramatically inflated (90%+ for PDM-accepted deals) and the CFO suspects cherry-picking rather than real PDM contribution.
**Solution:** PDM-accepted deals are almost always late-stage, high-quality opportunities where the sales rep already had strong conviction — this creates survivorship bias. Fix by running propensity-score matching: compare co-sell deals only against direct deals in the same stage, same ICP tier, and same ACV band at the point the ACE submission was made (not at close). A 15-25% win rate lift after matching is defensible; 40%+ without matching is cherry-picking optics.

**Problem:** Marketplace GMV number looks great but the CRO says most of it is just existing customers routing renewals through marketplace for EDP burndown — not real growth.
**Solution:** This is the most common cloud marketplace attribution trap. Implement the three-bucket classification immediately: (1) Net-new marketplace-sourced, (2) Co-sell hybrid, (3) EDP/MACC routing. Report all three separately. EDP burndown routing IS valuable — it protects NRR and prevents churn by making your product easier to buy — but it should not be counted as new logo ARR. Present the 3-bucket view to the CRO with separate growth targets for each bucket.

**Problem:** AWS PDM accepted 12 ACE opportunities last quarter but only showed up to 2 customer calls and added no visible deal velocity. How do we measure and report this diplomatically?
**Solution:** Build the PDM Contribution Score (events logged on opportunity after PDM assignment: customer meetings, intro emails, executive sponsor connection, marketplace deal support) vs. pipeline value accepted. Low-contribution PDMs typically accepted 8+ opportunities but logged 0-1 activities. Use the QBR to present this as "here's where we need more joint customer engagement to maximize our co-sell investment" rather than "your PDM isn't helping." Flag high-activity PDMs for relationship investment and request a PDM replacement or supplemental resource from your AWS PAM for low-activity territories.
