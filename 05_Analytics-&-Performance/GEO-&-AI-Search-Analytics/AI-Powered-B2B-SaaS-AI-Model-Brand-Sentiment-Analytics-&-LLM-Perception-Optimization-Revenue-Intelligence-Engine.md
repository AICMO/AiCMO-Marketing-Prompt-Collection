## Troubleshooting

**Problem: AI model responses are completely inconsistent across testing sessions, making CAVS scoring unreliable**
Solution: Standardize testing by using the exact same system prompt, temperature settings (if using API), and time of day. Run 3 tests per query per model and use the median response for scoring. Models with retrieval-augmentation (Perplexity, ChatGPT with Browsing) are highly dynamic — test weekly and report rolling 4-week averages to smooth out variance.

**Problem: AI models consistently describe the brand inaccurately (wrong use cases, outdated information, or competitor confusion)**
Solution: Inaccuracies in pure LLM outputs trace back to training data. Fix this by: (1) Publishing a comprehensive, clearly labeled "About [Company]" page with structured schema markup — this becomes a high-quality training signal; (2) Creating Wikipedia and Wikidata entries with verified, cited facts; (3) Ensuring Crunchbase, LinkedIn, and all third-party profiles are consistent and current. For retrieval-based inaccuracies (Perplexity, ChatGPT Browsing), prioritize updating the highest-authority pages that AI systems retrieve from.

**Problem: Revenue attribution from AI search is impossible to measure — buyers don't remember or report how they found the company**
Solution: Accept that 100% accurate attribution is impossible. Instead, build a triangulated model: (1) Direct: Self-reported form field + sales call logging (aim for 60-70% coverage); (2) Indirect: Track branded dark search (direct + no-referrer traffic) as a proxy — when this grows after AI program investments, attribute a portion to AI; (3) Statistical: Implement a clean A/B geographic test with AI program vs. control markets over 2 quarters to calculate incremental pipeline lift.
