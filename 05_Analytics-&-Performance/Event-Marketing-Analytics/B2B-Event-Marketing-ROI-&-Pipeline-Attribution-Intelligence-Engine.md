## Integration Tips
- **Salesforce / HubSpot:** Tag all event-sourced leads with campaign source at entry point (not just "event" — use "FinovateSpring-2025-Booth" or "NYC-Roundtable-Oct-2025"). Build a campaign influence report filtered by event campaign type to pull the influenced pipeline numbers directly. This eliminates manual data assembly for the Advanced prompt.
- **Marketo / Pardot / HubSpot Campaigns:** Use event attendance as a behavioral trigger in nurture programs — segment attendees into high-intent (requested demo, engaged 3+ times) vs. low-intent (scanned badge, no follow-up action) and feed the high-intent segment to SDRs within 24 hours; automate low-intent into 6-touch nurture sequence.
- **Splash / Eventbrite / Cvent:** Export attendee data with company name, title, and any session attendance signals. Run through ZoomInfo or Clearbit to append firmographic data and calculate ICP hit rate automatically before inputting to this prompt.
- **Clari / Gong:** Pull "event mentioned" deal stage from Clari pipeline data to identify deals where the event was discussed in sales calls — this surfaces pipeline acceleration that CRM tagging misses and improves attribution accuracy.
- **Google Sheets / Notion:** Build a rolling Event Performance Tracker template with columns matching this prompt's input fields. After each event, have marketing ops fill it in within 5 business days. Feed the accumulated tracker into this prompt quarterly for portfolio analysis without data hunting.
- **Zapier / Make:** Automate a post-event reporting workflow: (1) event ends → (2) Zapier pulls lead list from Cvent → (3) enriches via Clearbit → (4) pushes to HubSpot with event campaign tag → (5) triggers SDR task in Salesforce for ICP-fit contacts. Eliminates 4–6 hours of manual post-event ops per event.

## Troubleshooting

**Problem: CRM attribution is inconsistent — some event leads were tagged, others weren't, so pipeline numbers look artificially low.**
Solution: Use a two-step recovery method. First, pull the raw attendee list from your event platform and match it against CRM contacts by email (90% match rate) or company domain (catches +15% more). Second, filter CRM opportunities created within 90 days where the associated contact appears on the attendee list — this surfaces influenced pipeline that bad tagging missed. Flag the data quality gap in the prompt's attribution confidence field and use the recovered numbers as your analysis baseline while fixing tagging for future events.

**Problem: The report concludes I should cut a high-visibility event (e.g., a major industry conference) that leadership loves for brand reasons, not pipeline reasons.**
Solution: Separate the event's brand value from its pipeline ROI. Explicitly model two scenarios in your output: (A) current investment for pipeline, (B) reduced investment optimized for brand presence only (e.g., speaking slot + 2 staff, no booth = 70% cost reduction). Present the ROI of each scenario side-by-side. Most leadership teams will accept cutting booth spend when they see they can maintain brand visibility at 30% of the cost. Use the freed budget as the "reinvestment" story — show where it goes and what pipeline it generates instead.

**Problem: Executive roundtables and intimate events show great pipeline numbers but sample sizes are too small to be statistically reliable.**
Solution: Aggregate all similar-format events over 6–12 months before drawing conclusions. A single executive dinner with 3 opportunities could be an outlier; 12 dinners with 35 opportunities is a validated program. In the interim, use leading indicators (ICP hit rate, meeting-to-opportunity conversion rate, attendee seniority) as quality proxies while pipeline attribution matures. Flag in the analysis: "Program ROI confidence: Medium — based on N=4 events; recommend full analysis after N=12."
