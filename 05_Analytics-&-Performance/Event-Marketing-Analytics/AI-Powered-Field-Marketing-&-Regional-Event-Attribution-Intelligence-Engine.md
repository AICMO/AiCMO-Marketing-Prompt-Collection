## Integration Tips

- **Salesforce:** Create a custom Campaign object for each field event; use Campaign Influence (Customizable Attribution) with a "Field Event" touch type to capture W-shaped attribution automatically. Set up a Flow to auto-create CampaignMember records from event check-in lists via Zapier or Eventbrite API.
- **HubSpot:** Use Deal pipelines with a custom "Event Influenced" property; configure automated workflows that tag deals when an associated contact attended an event within 90 days prior to deal creation. Attribution reporting via the Revenue Attribution report with custom Interaction Types.
- **Marketo:** Create Programs for each event with Success Criteria defined as "Attended"; use Bizible/Marketo Measure's custom touchpoints to capture field event touches in multi-touch attribution waterfalls. Set Program Cost entries to enable ROI reporting.
- **Notion/Airtable:** Build a Field Marketing Command Center database with event records, attendee lists (linked to account records), pipeline outcomes, and auto-calculated ROI fields. Feed Claude this structured data quarterly for instant attribution reports.
- **Zapier/Make:** Automate the post-event workflow — Eventbrite check-in → Salesforce CampaignMember creation → Outreach/Salesloft sequence enrollment → Slack alert to field rep — all within 2 hours of event end.
- **Google Sheets:** Use the Advanced Version prompt output to populate a quarterly field marketing attribution model; connect via Salesforce exports and use VLOOKUP/QUERY to cross-reference attendee lists with opportunity data automatically.
- **Looker/Tableau:** Build a Field Marketing ROI dashboard using event-touched opportunity data with pre-built dimensions for territory, event type, account tier, and time-to-close. The prompt output provides the data model schema.

## Troubleshooting

**Problem: CRM data is incomplete — many event attendees don't have records or opportunities.**
Solution: Run the Quick Version prompt with whatever data you have, and explicitly note "X% of attendees lacked CRM records." The prompt will flag this as a confidence caveat and still generate directional ROI estimates. To prevent future data gaps, implement a post-event enrichment workflow: scan badge data or sign-in sheets against Apollo.io or Clearbit to auto-create CRM contacts within 48 hours of every event, then trigger an SDR sequence for all new contacts within that window.

**Problem: Attribution model shows conflicting results — last-touch gives events no credit, first-touch over-attributes early content.**
Solution: This is expected with any single-touch model for relationship-driven events. Switch to the W-shaped or full-path model as described in the Advanced Version. If your attribution tool doesn't support W-shaped, run a parallel analysis: compare 90-day pipeline and win rates for event-touched vs. non-event-touched accounts in the same territory and ICP cohort. The lift data tells the real ROI story independent of attribution model debates.

**Problem: Field events show strong pipeline but deals aren't closing — finance is skeptical.**
Solution: Reframe from pipeline to velocity and win rate lift. Calculate: "Without field events, our Q1 pipeline would have taken X additional days to progress, costing $Y in delayed revenue recognition." Use the Velocity Impact Analysis output from this prompt to quantify the acceleration value. Also build a 12-month rolling view — event-influenced revenue often lags 6-9 months for enterprise deals; show the trailing pipeline chart to demonstrate compounding returns.
