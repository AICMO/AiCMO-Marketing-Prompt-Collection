## Integration Tips

- **Microsoft Ads + HubSpot:** Enable the Microsoft Ads native HubSpot integration to sync MQL and pipeline data back into Campaign Manager for automated bidding on offline conversions. Use this data in the Module 1 Revenue Impact Scorecard for more accurate pipeline ROAS calculation.
- **Microsoft Clarity:** Install Clarity (free Microsoft heatmap/session recording tool) on all landing pages receiving Microsoft Ads traffic. Feed scroll depth, click heatmap, and rage-click data into Module 2's Landing Page Experience diagnosis.
- **Google Ads Import:** Use Microsoft Ads' native Google Ads import feature to maintain campaign parity, but override bids — Microsoft's auction dynamics often favor lower bids than Google's equivalent campaigns. Feed imported campaign structure into Module 3's cross-channel analysis.
- **Salesforce + Microsoft Ads Offline Conversion Tracking:** Use Microsoft's UET (Universal Event Tracking) tag with Salesforce webhook to pass Closed-Won revenue back into Campaign Manager as offline conversions. This enables Target ROAS bidding against actual closed revenue — the most accurate signal for B2B pipeline optimization.
- **Power BI / Looker Studio:** Export Microsoft Ads performance data via the API and build a unified paid media dashboard combining Microsoft Ads + Google Ads metrics. Feed this dashboard output into the prompt's performance data input section for automated weekly analysis.
- **Zapier/Make Automation:** Set up automated weekly data exports from Microsoft Ads Campaign Manager → Google Sheets → trigger this prompt via API for recurring Monday morning performance briefs.

## Troubleshooting

**Problem:** Quality Score data isn't appearing in the Microsoft Ads export.
**Solution:** Quality Score is only available at the keyword level, not ad group or campaign level. Export the Keywords report from the Reports tab (not the default Campaigns view), selecting "Quality Score," "Expected CTR," "Ad Relevance," and "Landing Page Experience" as columns. If Quality Score shows "N/A," the keyword has received fewer than 1,000 impressions in the past 30 days — exclude these from Module 2 analysis or use a 90-day window.

**Problem:** Cross-channel comparison with Google Ads shows wildly different CPLs for seemingly equivalent keywords.
**Solution:** Microsoft Ads typically reaches 20-30% older, higher-income demographics via Bing — this demographic composition difference (not ad quality) often explains CPL variance. Before declaring one platform superior, segment by ICP-match rate (lead score, job title, company size) not just raw CPL. Also verify that conversion tracking is implemented identically on both platforms — mismatched tracking is the most common cause of false CPL discrepancies.

**Problem:** LinkedIn Audience Network is showing high click volume but near-zero conversions.
**Solution:** Microsoft's LinkedIn Audience Network uses professional demographic data from LinkedIn to target display and native ads, but the placements are on third-party publisher sites — not LinkedIn.com. These placements often generate low-intent, high-bounce traffic. Exclude placements with bounce rate > 75% using the Placement Exclusions feature. Consider disabling Audience Network entirely for campaigns where Search CPL is on target — the incremental volume rarely justifies the CPL inflation for B2B SaaS.
