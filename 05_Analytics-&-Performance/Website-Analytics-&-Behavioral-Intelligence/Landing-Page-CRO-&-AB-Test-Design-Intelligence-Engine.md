## Integration Tips

**Optimizely / VWO / Convert:**
Export each experiment brief as a test card. Use the hypothesis field verbatim as the test description. Set the MDE and sample size targets directly in the tool's traffic allocation settings.

**Google Analytics 4:**
Create custom GA4 events for each test's primary and secondary metrics before launch. Use GA4's "Experiments" feature for server-side tests or integrate with your testing tool via GTM.

**HubSpot Landing Pages:**
For HubSpot-hosted pages, use the built-in A/B testing module for single-variable tests. Export winning copy variants to the HubSpot design manager for global templates.

**Salesforce / HubSpot CRM:**
Append UTM parameters with variant IDs (`utm_content=variant-a-headline`) so winning variant data flows into CRM and connects to downstream pipeline and closed-won revenue.

**Notion / Linear / Jira:**
Paste each experiment brief directly into your backlog. Use the S/M/L complexity rating to assign sprint points. The revenue impact calculation becomes the "business value" field for prioritization.

**Zapier / Make Automation:**
Build a workflow: when a test reaches statistical significance in your CRO tool → auto-create a Jira ticket with the winning variant, lift achieved, and revenue impact → notify the team in Slack → add to a "wins library" Notion database.

**Hotjar / FullStory / Microsoft Clarity:**
After running each test, segment session recordings by variant to watch qualitative behavior changes. Often reveals why a variant won and surfaces the next test hypothesis.

## Troubleshooting

**Problem: "The AI gives generic CRO advice like 'add social proof' without specificity."**
Fix: Include more data in your input. Paste the exact headline copy, describe the exact social proof present, and share specific heatmap or session recording observations. The more evidence you provide, the more specific the diagnosis. Generic inputs produce generic outputs.

**Problem: "Traffic is too low to reach statistical significance in a reasonable time."**
Fix: Add the traffic constraint to the prompt explicitly: "We have 2,000 visitors/month — only recommend tests with a 10%+ MDE that can be validated in 4-6 weeks." The prompt will shift toward high-impact, high-MDE tests (major redesigns, big copy swaps) instead of small element tests.

**Problem: "We ran the top recommended test and it lost — CVR went down."**
Fix: This is valuable data. Feed the losing result back into the prompt: "Test A (headline change to X) reduced CVR by 8% — why might this have happened, and what should we learn for the next hypothesis?" Use losers as a diagnostic tool. A -8% result on relevance tells you the existing headline was doing more work than the LIFT audit suggested.
