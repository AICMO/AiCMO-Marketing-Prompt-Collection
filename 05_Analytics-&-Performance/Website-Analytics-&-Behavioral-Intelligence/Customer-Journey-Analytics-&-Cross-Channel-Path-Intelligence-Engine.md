## Overview
Transforms raw multi-channel behavioral data into a complete map of how buyers actually move from awareness to purchase — identifying the highest-converting paths, hidden friction points, and journey shortcuts that unlock revenue. Use it when you need to understand the true buyer journey (not the one your team assumed) and build a prioritized roadmap to accelerate path-to-purchase across every channel.

## Integration Tips

- **GA4**: Use the "Path Exploration" report (Explore tab) to export the top 50 most common user paths; feed directly into the Quick Copy-Paste prompt. Enable Google Signals for cross-device path stitching.
- **Mixpanel / Amplitude**: Use the "Flows" or "Pathfinder" report to export converting sequences; these tools provide richer behavioral sequence data than GA4 for product-led journeys.
- **HubSpot**: Pull "Contact Activity Timeline" for won deals — aggregate 20-30 won deals' journey touchpoints to identify the real sales-cycle patterns; paste summary into the Advanced Version.
- **Salesforce + Pardot/Marketing Cloud**: Use "Campaign Influence" reports to identify which marketing touchpoints appear most frequently in closed-won opportunities; map these to the Channel Sequencing Strategy section.
- **Heap / FullStory**: Export "Most Common Paths to Conversion" from Heap's auto-capture data; this provides granular in-session click sequences that reveal micro-conversion patterns.
- **Looker Studio / Tableau**: Build a journey dashboard pulling GA4 paths + CRM deal stage data; run this prompt monthly against updated dashboard exports.
- **Zapier / Make**: Automate weekly path data exports from GA4 → Google Sheets → trigger this prompt via Claude API to generate a standing weekly journey intelligence report.
- **Segment CDP**: Use Segment's "Journeys" feature to define the paths identified in this analysis, then activate personalized messaging for users entering high-converting paths.

## Troubleshooting

**Problem: Journey path data shows "direct" as the dominant channel, making sequences impossible to interpret.**
Solution: "Direct" in GA4 almost always means the previous session's source was lost — this is the dark funnel problem. Use UTM parameters religiously across all paid channels, implement GA4 server-side tagging to reduce session fragmentation, and cross-reference with your CRM's "Original Source" field (which is set at contact creation and doesn't reset). Run the prompt with CRM data as primary source rather than GA4 when direct dominates.

**Problem: The top converting paths have too few data points (under 50 conversions/month) to make statistically reliable recommendations.**
Solution: Aggregate data over a 90-day window rather than 30 days; group similar channels (e.g., all paid social together) rather than individual platforms; focus the analysis on funnel stage patterns (awareness → consideration → decision) rather than exact channel sequences. Flag low-volume paths in your output and treat recommendations as directional hypotheses requiring A/B validation rather than proven insights.

**Problem: The analysis identifies a Golden Path that requires channels I don't currently have budget for.**
Solution: Use the revenue-weighting math to build a budget case — if the G2 → Pricing → Demo path generates $515K in annualized revenue on current organic traffic, calculate what a 3× increase in G2 traffic (via G2 review generation program at $X cost) would generate. Present this as a specific ROI-positive investment to leadership rather than a channel request. Focus 30-day quick wins on improving conversion on paths you already have volume in before requesting new budget.
